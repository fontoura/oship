OSHIP FAQ

  1. What do I need to know to use OSHIP for my application?

    You will need very basic web development knowledge to get started. It helps to have some knowledge of Python, a templating language and of course the openEHR model. Most complex applications will require a team of people with various skills. However, the OSHIP community stands ready to help you transition to future proof healthcare application development.

  2. Is OSHIP robust and scalable?

    OSHIP uses the very same core technologies that Zope Corporation and thousands of consultants and developers use for very large web development applications. While we do not specifically use the Plone CMS we do share the same underlying functionality. We do use the Grok development environment to ease the learning curve of the Zope Component Architecture.

  3. Can I use existing databases in combination with an openEHR application?

    Yes. There are adapters for most popular SQL databases and you can mix in that data with your openEHR based information as needed.

  4. I have an existing XMLSchema model. Can I use this with OSHIP?

    Yes. There are even methods to help you bootstrap your model into an OSHIP model using standard UML tools.
